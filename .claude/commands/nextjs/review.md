# Claude 명령어: Review

현재 변경사항에 대한 코드 리뷰를 수행합니다.

## 사용법

```
/nextjs:review
/nextjs:review --focus=security
/nextjs:review --focus=performance
```

## 인자

$ARGUMENTS - 리뷰 옵션 (선택사항)
- `--focus=security`: 보안 취약점 집중 검토
- `--focus=performance`: 성능 최적화 집중 검토
- `--focus=accessibility`: 접근성 집중 검토
- 옵션 없음: 전체 검토

## 프로세스

1. `git status`로 변경된 파일 확인
2. `git diff`로 스테이지된/스테이지되지 않은 변경사항 분석
3. 변경된 파일 내용 읽기
4. 체크리스트 기반 리뷰 수행
5. 리뷰 결과 요약 제공

## 리뷰 체크리스트

### TypeScript 타입 안정성
- [ ] `any` 타입 사용 여부 확인
- [ ] 적절한 타입 정의 확인
- [ ] 타입 단언(`as`) 남용 여부
- [ ] null/undefined 처리 확인

### React/Next.js 베스트 프랙티스
- [ ] 훅 규칙 준수 (조건부 호출 금지)
- [ ] 의존성 배열 정확성
- [ ] 불필요한 리렌더링 방지
- [ ] 'use client' / 'use server' 적절한 사용
- [ ] 메타데이터 정의 확인

### 보안 취약점
- [ ] XSS 취약점 (dangerouslySetInnerHTML)
- [ ] 사용자 입력 검증
- [ ] 민감 정보 노출 (API 키, 비밀번호)
- [ ] CSRF 보호
- [ ] SQL 인젝션 가능성

### 성능 최적화
- [ ] 불필요한 상태 관리
- [ ] 메모이제이션 필요 여부 (useMemo, useCallback)
- [ ] 이미지 최적화 (next/image 사용)
- [ ] 동적 임포트 활용
- [ ] 번들 크기 영향

### 접근성 (a11y)
- [ ] 이미지 alt 속성
- [ ] aria 속성 적절성
- [ ] 키보드 네비게이션
- [ ] 색상 대비
- [ ] 시맨틱 HTML 사용

### 코드 품질
- [ ] 네이밍 컨벤션 준수
- [ ] 중복 코드 여부
- [ ] 함수/컴포넌트 크기 적절성
- [ ] 에러 처리 적절성
- [ ] 주석 필요 여부

## 리뷰 결과 포맷

```markdown
## 코드 리뷰 결과

### 요약
- 검토 파일: N개
- 발견된 이슈: N개 (심각: N, 경고: N, 정보: N)

### 심각 (즉시 수정 필요)
1. **[파일:라인]** 이슈 설명
   - 문제점: ...
   - 해결방안: ...

### 경고 (권장 수정)
1. **[파일:라인]** 이슈 설명
   - 문제점: ...
   - 해결방안: ...

### 정보 (개선 제안)
1. **[파일:라인]** 이슈 설명
   - 제안: ...

### 잘한 점
- ...
```

## 참고사항

- 스테이지된 변경사항 우선 검토
- 변경사항이 없으면 알림
- `/commit` 전에 실행 권장
- 발견된 이슈는 심각도별 분류
